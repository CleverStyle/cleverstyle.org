HybridAuth module, adds functionality for integration with external services for login and more

Module is based on HybridAuth library, so, refer to hybridauth.sourceforge.net for specific details of this library.

To get instance of \Hybrid_Auth class with current configuration \cs\modules\HybridAuth\get_hybridauth_instance() function is used.

Example of getting Mark Zuckerberg profile on facebook:

$HybridAuth		= \cs\modules\HybridAuth\get_hybridauth_instance();
try {
	if (
		$HybridAuth->isConnectedWith('Facebook') &&
		($api = $HybridAuth->getAdapter('Facebook')->api())
	) {
		$mark_profile	= $api->api('zuck');
	}
} catch (Exception $e) {}

Also this module have several triggers:

  • HybridAuth/registration/before

    Executes before registration of new user by this module

    [
	'provider'	=> provider
	'email'		=> email
	'identifier'	=> identifier
	'profile'	=> profile
]
    provider
    Provider name
    email
    Email
    identifier
    Identifier given by provider
    profile
    Profile url

    Third-party component may register trigger to know, when new user is registered by HybridAuth module

  • HybridAuth/add_session/before

    Executes before creation of new session by this module

    [
	'adapter'	=> $Adapter
	'provider'	=> provider
]
    $Adapter
    instance of Hybrid_Provider_Adapter
    provider
    Provider name

    Third-party component may register trigger to know, when new user session created by HybridAuth module

  • HybridAuth/add_session/after

    Executes after creation of new session by this module

    [
	'adapter'	=> $Adapter
	'provider'	=> provider
]
    $Adapter
    instance of Hybrid_Provider_Adapter
    provider
    Provider name

    Third-party component may register trigger to know, when new user session created by HybridAuth module
