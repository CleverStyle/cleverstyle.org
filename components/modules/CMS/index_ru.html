CleverStyle CMS

CleverStyle CMS это простая, масштабируемая, и быстрая система управления контентом.

Текущая версия Alpha.

Ядро системы содержит только базовую функциональность, поэтому больше похоже на CMF (фреймворк управления контентом) чем на CMS.

Но паралельно с ядром разрабатываются некоторые компоненты:

Модули

  • Blogs (простая блоговая функциональность)
  • Comments (добавляет функциональность комментариев в другие модули)
  • Cron (простой графический интерфейс для crontab, запланированных задач)
  • Deferred tasks new (отложенные задачи позволяют другим компонентам создавать задачи, которые могут быть выполнены не немедленно, а немного позже)
  • Disqus (интеграция сервиса комментариев Disqus, добавляет функциональность комментариев в другие модули)
  • Feedback (простой модуль обратной связи, отправляет сообщение на email администратора)
  • HybridAuth (интеграция библиотеки HybridAuth для интеграции с социальными сетями и дрругими сервисами)
  • OAuth2 (обеспечивает реализацию протокола авторизации OAuth 2 (серверная сторона))
  • Photo gallery (простой модуль фотогалереи, который базируется на компонентах Fotorama, Plupload и SimpleImage)
  • Plupload (интеграция Plupload для загрузки файлов, добавляет функциональность загрузки файлов в другие модули)
  • Static pages (позволяет создавать простые статические страницы, например О Нас, или страницы без интерфейса, например для подтверждения владения сайтом)

Плагины

  • Admin default theme (принудительно установит тему CleverStyle в администрировании, что бывает полезно, так как не нужно адаптировать тему для админ панели)
  • Fotorama (интеграция jQuery галереи Fotorama в CleverStyle CMS)
  • SimpleImage (интеграция класса SimpleImage в CleverStyle CMS)
  • TinyMCE (интеграция TinyMCE WYSIWYG Редактора для обеспечения простого редактирвоания контента)

Основные возможности

  • Компоненты
    • Модули (для отображения основного содержания страницы)
    • Плагины (загружаются на каждой странице и обеспеивают дополнительную функциональность)
    • Блоки (размещаются вокруг страницы для отображения дополнительной информации)
  • Человекочитаемые адреса
  • Типы и группы пользователей
  • Права и контроль доступа
  • Локализация и интернационализация (как интерфейса, так и контента)
    • Автоматический перевод контента
  • Минификация и компрессия CSS и JavaScript
  • Зеркала сайта
    • Зеркала доменов
    • Зеркала физических серверов для каждого домена
  • Темы и цветовые схемы
  • Множественные базы данных
    • Множественные зеркала баз данных
  • Множественные хранилища файлов
  • Системное кэширование (FileSystem, APC, Memcached)
  • Фильтр и ограничения по IP
  • RESTful API
  • IPv6

Требования

  • Unix-подобрая операционная система или Windows (недостаточно тестируется, но всё равно должно работать)
  • Веб-сервер Apache2 или Nginx
  • PHP 5.4+
    • Библиотека mcrypt (рекомендуется для шифрования)
    • Библиотека cUrl (рекомендуется для автоматических переводов)
    • APC (Alternative PHP cache) модуль (рекомендуется для ускорения системы с PHP 5.4)
    • Memcached (опционально для Memcached движка кэша)
  • MySQL сервер баз данных (MariaDB тоже будет работать)
    • Система может быть расширена для поддержки других серверов баз данных
